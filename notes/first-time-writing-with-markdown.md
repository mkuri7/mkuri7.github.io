# Markdownで記事を書くという小さな一歩

公開日：2026年7月2日

今日は、GitHub Pagesでブログを書くために、Markdownという形式を意識して使い始めました。

齢56歳にして初めて！！ｗｗｗｗ

Markdownという言葉は以前から知っていましたが、実際に自分の記事を書くための道具として使うのは、今回がほぼ初めてです。

最初は少し戸惑いましたが、まぁHTMLの簡易版みたいなもんかな。文章を構造化するのに簡単でよいですね。

普通の文章を書く感覚とは違い、見出しには `#` を使い、箇条書きには `-` を使い、リンクやコードも決まった書き方があります。
けれども、慣れてくると、これは単に文章を書くための軽いルールなのだと感じました。

Wordのように見た目を細かく整えるのではなく、文章の構造を素直に書いていく。
タイトル、見出し、本文、リスト、引用、コード。

その一つひとつを、シンプルな記号で表現していくところが面白いです。

今回、Masao’s LabをGitHub Pagesで始めたことで、文章を書くことと、少しだけ技術に触れることが自然につながりました。

これは大きな成果ではありません。
でも、50代になってから新しい道具を覚え、自分の言葉をWebに残していくための小さな一歩です。

これからは、英語学習、Python、AI、仕事、日々の暮らしの記録を、このMarkdownというシンプルな形式で少しずつ残していきたいと思います。

> 小さな記号で、考えを形にしていく。
