# MASAO Learning Roadmap

Version 1.0

This document defines the long-term learning roadmap for MASAO in the AI era.

The objective is **not to master programming languages**, but to become an **AI Product Engineer** capable of designing and building practical software products.

---

# Core Philosophy

Programming languages are tools.

Products are the goal.

In the past, software engineers were evaluated by how well they could write code.

Today, the more valuable skill is designing products, selecting appropriate technologies, leveraging existing components, and using AI to accelerate implementation.

Learning should therefore be driven by **what you want to build**, not by collecting programming languages.

---

# The AI Product Engineer

The target is **not** to become a traditional full-stack engineer.

The target is to become an **AI Product Engineer**.

Responsibilities include:

- Identifying real problems
- Designing simple and maintainable architectures
- Selecting the right technologies
- Leveraging AI effectively
- Combining reusable libraries and components
- Building beautiful and practical products

Programming remains important.

However, programming is now only one part of product engineering.

---

# Learning Roadmap

## Phase 1 — Foundation

Build a strong foundation for modern software development.

### Python ★★★★★

Primary development language.

Learn:

- Python fundamentals
- uv
- Virtual environments
- pathlib
- SQLite
- requests
- pytest
- ruff
- Git integration

Primary use cases:

- LotoLab AI
- Alfred Library
- CLI applications
- AI utilities

---

### Git ★★★★★

Essential development tool.

Learn:

- Commit
- Branch
- Merge
- GitHub workflow

---

### AI Tools ★★★★★

The most important technology.

Become proficient with:

- ChatGPT
- Claude
- GitHub Copilot
- Cursor
- Prompt Engineering

The ability to collaborate with AI will become one of the strongest technical skills.

---

# Phase 2 — Frontend Development

Learn enough frontend technology to build modern user interfaces.

## JavaScript ★★★★☆

Do not attempt to memorize the entire language.

Focus on:

- import / export
- Objects
- Arrays
- Arrow functions
- async / await

---

## React ★★★★★

React is primarily a component system.

The goal is not writing React from scratch.

The goal is understanding and using components.

Examples:

- RichText
- TextControl
- InspectorControls
- PanelBody

Eventually:

```jsx
<ProductCard />
```

should feel as natural as calling a Python function.

---

## WordPress Block Development ★★★★★

Primary technology for Alfred Affiliate.

Learn:

- block.json
- Gutenberg Blocks
- Attributes
- Inspector Controls
- Rendering
- Components

---

## CSS ★★★☆☆

Enough to build beautiful interfaces.

Focus on:

- Flexbox
- Grid
- Typography
- Border Radius
- Shadows
- Spacing

Good design matters.

---

# Phase 3 — Product Engineering

This is the most important phase.

Programming languages become secondary.

Design becomes primary.

---

## Software Design ★★★★★

Learn how software should be structured.

Topics include:

- Separation of concerns
- Product Data
- Renderer
- Generator
- Library design
- Reusable components
- Maintainability

Software Design is becoming more valuable in the AI era because AI can generate code, but it cannot replace thoughtful architecture.

---

## UI / UX Design ★★★★★

Beautiful products create better experiences.

Follow the principles defined in MOM.

- Design Matters
- Beautiful by Default
- Reduce friction
- Keep interfaces calm and intuitive

---

## Product Architecture ★★★★★

Think in systems rather than individual files.

Example:

```
Generator

↓

Product Library

↓

Renderer

↓

WordPress
```

Architecture is one of the few things AI cannot fully automate.

---

# Phase 4 — AI Development

Expand into AI-powered applications.

Learn:

- OpenAI API
- Anthropic API
- MCP
- AI Agents

These technologies will support future Alfred products.

---

# Phase 5 — Additional Technologies

Learn technologies only when they become necessary.

## TypeScript ★★★★★

Highest priority after JavaScript.

Reason:

React and modern frontend development increasingly rely on TypeScript.

Learn it naturally while building products.

---

## PHP ★★★☆☆

Only enough to support WordPress development.

---

## Swift ★★☆☆☆

Learn when a Mac or iPhone application becomes necessary.

The goal is not mastering Swift.

The goal is quickly building useful native applications.

---

## Go ★★☆☆☆

Useful for fast CLI tools.

Currently lower priority than Python.

---

## Rust ★☆☆☆☆

Interesting technology.

Not currently aligned with long-term product goals.

---

# Technologies That Do Not Require Deep Study

There is no need to become an expert in every language.

Examples include:

- Java
- C#
- C++
- Kotlin
- Ruby
- Perl

Existing knowledge remains valuable.

Deep specialization is unnecessary unless required by a future product.

---

# Reference-Driven Learning

Instead of memorizing syntax,

build a personal development reference.

Example:

```
MASAO Development Reference

Python

React

WordPress

TypeScript

Docker

Git

AI Prompt Engineering

UI Components

Design Patterns
```

This is not a language manual.

It is a practical reference describing:

- What a component does
- When to use it
- Common examples
- Best practices

Over time this becomes a reusable engineering knowledge base.

---

# Product-Driven Learning

Never ask:

> What programming language should I learn next?

Instead ask:

> What product do I want to build?

The product determines the technology.

Examples:

LotoLab AI

↓

Python

---

Alfred Affiliate

↓

React

TypeScript

WordPress

---

Native macOS Utility

↓

Swift

SwiftUI

Learning becomes purposeful rather than theoretical.

---

# Final Vision

The long-term objective is not to become a programmer who knows many languages.

The objective is to become someone who can:

- Design useful products
- Build them quickly with AI
- Reuse high-quality components
- Deliver beautiful user experiences
- Solve real problems

Programming languages are tools.

Products are the destination.
