# VS Codeで最初に覚えるお作法

VS Codeでは、すべての文字を一字ずつ手入力する必要はありません。

大切なのは、コードの意味や構造を自分で考えながら入力し、スペル、閉じ括弧、インデントなどの機械的な作業をVS Codeに任せることです。

基本的な考え方は次のとおりです。

```text
自分で意図して入力を始める
↓
補完候補を確認する
↓
Tabで確定する
↓
最後にフォーマッターで整える
```

## 1. すべてを最後まで手入力しない

例えば、次の途中まで入力します。

```text
classN
```

補完候補に `className` が表示されたら、`Tab` または `Enter` で確定します。

```jsx
className
```

これは手抜きではなく、VS Codeを使った標準的な入力方法です。

補完を利用すると、次のようなミスを減らせます。

- スペルミス
- 大文字と小文字の間違い
- 存在しない変数名の入力
- ReactやWordPress APIのプロパティ名の間違い

## 2. 補完候補は `Control + Space` で表示する

補完候補が自動的に表示されない場合は、次のキーを押します。

```text
Control + Space
```

例えば、次の位置で補完候補を表示します。

```jsx
<img
  s
```

状況に応じて、`src` などの候補が表示されます。

macOSの入力ソース切り替えと競合する場合は、Command Paletteから次のコマンドを実行します。

```text
Trigger Suggest
```

## 3. HTMLやJSXのタグはEmmetを利用する

VS Codeには、短い省略記法からHTMLやJSXを生成するEmmetがあります。

例えば、次を入力します。

```text
div.alfred-amazon-card__image
```

Emmetの候補を選択して `Tab` を押すと、次のコードに展開されます。

```jsx
<div className="alfred-amazon-card__image"></div>
```

同様に、次のような入力も利用できます。

```text
span
```

```jsx
<span></span>
```

```text
img
```

```jsx
<img src="" alt="" />
```

候補が複数表示された場合は、`Emmet Abbreviation` と表示された候補を選びます。

## 4. 閉じタグはVS Codeに任せる

開始タグを入力すると、多くの場合は閉じタグも自動的に追加されます。

```jsx
<div></div>
```

カーソルは開始タグと閉じタグの間に配置されます。

```jsx
<div>|</div>
```

ここで `Enter` を押すと、次のように展開されます。

```jsx
<div>
  |
</div>
```

閉じタグが自動生成されている場合は、自分でもう一度入力しないようにします。

閉じタグの二重入力は、JSXの構造が崩れる原因になります。

## 5. `Tab` と `Enter` を使い分ける

補完候補やスニペットを操作するときは、次の使い分けを基本とします。

- `Tab`：候補を確定し、次の入力位置へ移動する
- `Enter`：通常の改行、または補完候補の確定に使う
- `Esc`：不要な補完候補を閉じる

特にEmmetやスニペットでは、`Tab` を押すことで入力位置を順番に移動できます。

```jsx
<img
  src=""
  alt=""
/>
```

## 6. 括弧や引用符は片側だけ入力する

VS Codeは通常、次の文字を自動的に閉じます。

```text
(
{
[
"
'
```

例えば、`{` を入力すると、対応する `}` も自動的に入力されます。

```jsx
{ | }
```

同様に、`(` を入力すると `)` が追加されます。

```jsx
( | )
```

入力中は、VS Codeが作成した閉じ括弧や引用符を確認し、同じ文字を重複して入力しないようにします。

今回のような条件付き表示では、括弧の対応関係を意識します。

```jsx
{ imageUrl ? (
  <img />
) : (
  <span>No Image</span>
) }
```

## 7. インデントはフォーマッターに任せる

入力中にインデントが多少崩れても、手作業で完全に整える必要はありません。

ファイル全体を整形するMacのショートカットは次のとおりです。

```text
Shift + Option + F
```

Command Paletteから実行する場合は、次のコマンドを選びます。

```text
Format Document
```

選択した範囲だけを整形する場合は、次のコマンドを使用します。

```text
Format Selection
```

JSXでは括弧やタグが多いため、人間が空白を数えるよりもフォーマッターを利用する方が安全です。

基本的には、コードを変更した後に一度フォーマットします。

## 8. 行の移動は切り取りと貼り付けを使わない

現在の行や選択した行を上下へ移動する場合は、次のショートカットを使います。

```text
Option + ↑
Option + ↓
```

行を複製する場合は、次のショートカットを使います。

```text
Shift + Option + ↓
```

似たコードを追加するときに便利です。

## 9. 同じ名前を変更するときは複数選択を利用する

現在選択している文字列と同じ文字列を、次の位置でも選択する場合は次を使います。

```text
Command + D
```

例えば、`imageUrl` を選択して `Command + D` を押すと、次の `imageUrl` も選択されます。

ファイル内の同じ文字列をすべて選択する場合は次を使います。

```text
Shift + Command + L
```

任意の位置へ複数のカーソルを追加する場合は次を使います。

```text
Option + Click
```

慣れるまでは、`Command + D` で一つずつ確認しながら選択する方が安全です。

## 10. Command Paletteを活用する

VS Codeのすべてのショートカットを最初から暗記する必要はありません。

Command Paletteは次のキーで開きます。

```text
Shift + Command + P
```

Command Paletteでは、実行したい機能名を検索できます。

```text
Format Document
Format Selection
Trigger Suggest
Rename Symbol
Organize Imports
Emmet
```

最初はショートカットよりも機能名を理解し、頻繁に使う機能だけを徐々に覚えます。

## 11. 入力後はエラー表示を確認する

VS Codeは、問題があるコードに赤色や黄色の波線を表示します。

- 赤い波線：構文エラーや明確な問題
- 黄色い波線：警告や改善候補
- 灰色表示：未使用の変数やimport

波線にマウスポインターを合わせると、問題の内容が表示されます。

Macでは次のキーで、現在のファイル内にある問題へ移動できます。

```text
F8
```

前の問題へ戻る場合は次を使います。

```text
Shift + F8
```

エラーが出た場合は、すぐにコード全体を書き直すのではなく、波線が表示された位置と、その直前の括弧やタグを確認します。

## 12. ファイルを保存してビルド結果を確認する

コードを変更したら、次の順番で確認します。

1. ファイルを保存する
2. VS Codeの赤い波線がないことを確認する
3. `npm start` を実行しているターミナルを確認する
4. ビルドエラーがないことを確認する
5. ブラウザで実際の表示を確認する

コードが正しそうに見えても、ビルドと実際の画面の両方を確認します。

## JSXを効率的に入力する例

次のコードを入力する場合を考えます。

```jsx
<img
  src={ imageUrl }
  alt={ title || __( 'Product Image', 'alfred-affiliate' ) }
/>
```

効率的な入力手順は次のとおりです。

1. `img` と入力する
2. Emmet候補を `Tab` で確定する
3. `src` の入力位置へ移動する
4. `{` を入力する
5. `image` まで入力する
6. 補完候補の `imageUrl` を `Tab` で確定する
7. `alt` の入力位置へ移動する
8. `{ title || __(` まで入力する
9. 引用符と括弧の自動補完を利用する
10. 最後に `Shift + Option + F` で整形する

## 基本方針

VS Codeを使った入力では、次の役割分担を意識します。

### 自分で行うこと

- 何を実装するか考える
- コードの構造を理解する
- 適切な変数やコンポーネントを選ぶ
- 補完候補が正しいか確認する
- 実際の動作を確認する

### VS Codeに任せること

- スペルの補完
- タグや括弧を閉じる作業
- インデントの整形
- 同じ文字列の一括選択
- 構文エラーの検出
- 利用可能なプロパティや関数の候補表示

コードの意図と構造は自分で考え、機械的な作業はVS Codeに任せます。

これが、理解を深めながら入力ミスを減らすための基本的なお作法です。
